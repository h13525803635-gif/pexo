# 项目 23648081859：字幕超出安全区 — 原因与解决方案

**Langfuse trace**：`analysis/langfuse-data/cases/23648081859/trace-1-b1ca9a45.json`（约 29MB，为完整拉取的原始 trace。）  
**分析日期**：2026-04-24  

---

## 一、用户侧现象

- 成片为 **1:1（本例输出 960×960）** 的 e-learning 教学向视频，**底部英文字幕**在观感上**贴边、或超出平台/画幅「安全区」**（与画面底部 UI/装饰或播发端裁切冲突）。  
- 需求中要求 **matching English subtitles, easy to read**；实际装配使用 **`kind: "text"`** 烧录字幕。

---

## 二、从 trace 可核对的根因

### 1. 使用固定底边 `y: "h-130"`，未随画高与安全区比例留边

最终**成功**渲染的 `execute_edit_video` 中，每条 `text` clip 的 `text_opts` 均含：

- `"x": "(w-text_w)/2"`
- `"y": "h-130"`

在 **h=960** 时，约等于把字幕**锚在距底 130px** 的窄带。该数值**不随**输出分辨率、行数、背景框厚度变化，也**不等价**于各平台常用的 **8%～12% 画高** 的 action/title safe。  

### 2. 多行、长句、`box` + 粗描边 → 有效高度大于「130px 槽」

同批参数使用 `fontsize` 32～34、`box: "1"`、`boxborderw` 14～16、`borderw` 4 等。英文长句会**换行**，`text_h` 增大；在 **`y` 未按 `text_h` 上移** 的情况下，**整块**（字+描边+背景）容易**更贴近下沿**，视觉上即「超出安全区 / 出画」风险上升。

### 3. 与画内内容的安全区冲突（内容层）

本 case 画面为 **FlowSync 教程式 UI/卡片**；字幕始终压在**最底部**时，易与**画面内底部 UI/进度条**在**同一视带**叠加，从设计上也表现为「不该压字区域被占满」。

### 4. 与首次 `subtitle` 轨失败路径的关系（旁注）

曾尝试 `kind: "subtitle"` 等，出现过 `out_ms not set` 等 DSL 错误，后改为**手搓 `text` 轨**；**安全区问题的主因**仍是 **`text` 的 `h-130` 与内容形态**，与上述 error 不矛盾。

---

## 三、解决方案（可落地）

### 1. 改 `y` 表达式：禁止与画幅无关的常数

- **推荐**（若渲染链支持 `text_h`，与 FFmpeg drawtext 习惯一致）：

  - `"y": "h-text_h-h*0.10"`

  使字幕块**底侧**与画面下缘之间保留约 **10% 画高** 的安全带；多行时 `text_h` 增大，`y` 自动上移。  

- **若不支持 `text_h`**：用更大固定底边替代 `130`，如 **`h*0.18`～`h*0.22` 对应的像素** 作为最小留白（960 高约 **173～211px** 量级），并尽快产品侧支持 `text_h` 或统一安全区 API。

### 2. 长句策略

- 对超长 cue **拆成连续两条 `text` clip** 或略降 `fontsize`；  
- 略减 `boxborderw` / `borderw`，减少「可视块」比纯字号多占的高度。

### 3. 能走通时优先 `subtitle` 轨

若产品修复 `subtitle` + 时长探测问题，**优先**用 `margin_v` 等与规范对齐的烧录方式；在此之前 **`text` 轨必须**按第一节用比例/`text_h` 约束，**不要**再全局写死 `h-130`。

### 4. 写进 assembly-skill 的硬规则（摘要）

| 项目 | 规则 |
|------|------|
| 底边 | 禁止全项目固定 `h-130` 类与分辨率无关的像素；须 `h*0.08～0.12` 或 `h-text_h-h*0.10` |
| 1:1 / 竖版 | 底安全区可略宽于横屏（如从 10% 起） |
| 长句 | 拆条或降字号，避免两行仍压在固定窄底槽 |
