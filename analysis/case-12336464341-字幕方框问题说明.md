## 二、先分清：方框一般意味着什么

| 现象 | 常见技术含义 |
|------|----------------|
| **每个汉字都是同一占位框** | 当前用于绘字的 **字体面（font face）里没有该码点的字形**；渲染仍成功，但用 `.notdef` 之类占位符画出。 |
| **零星某几个字是框** | 多为该字在字体中未覆盖（生僻字、异体），或某段用了不同字体。 |
| **问号、菱形** | 更偏 **编码/解码** 链路问题；与本 case 的「满屏统一方框」典型画像不同。 |

本 case 属于 **第一行**：整条中文在画面上以方框替代。

## 三、从真实 trace 能确认什么（非猜测）

以下来自 Langfuse observation **`video-editor__execute_edit_video`**（`id: 12c0f31a57f7fc36`）及前序 **`list_fonts`**：

1. **入参文案正常**：`kind: "text"` 各 clip 的 `text` 为完整中文句（如「有些地方，只有仰头才能看见」等）。  
2. **字体选择在目录上合法**：`list_fonts` 返回中含 **`noto-serif-cjksc`**（Noto Serif CJK SC），`languages` 含 **`zh-Hans`**，`weights` 含 **300**。  
3. **实际调用与目录一致**：`text_opts` 使用 **`font: "noto-serif-cjksc"`**，**`font_weight: "300"`**（字符串）。  
4. **工具返回成功**：`success: true`，`statusMessage` 为空；**无** FFmpeg `stderr`、无解析后的物理字体路径回传。  

因此：**不是**「Agent 没写中文」或「误选了仅拉丁字体 id」这种表层错误；**也不是**「宋体系列本身不能显示中文」——Noto Serif CJK 即为简中排版设计。

## 四、根因归纳（执行端为何「对不上」字库）

trace **不能**直接给出「服务器加载了哪一个 `.ttf`」的日志，但结合 **入参正确 + 全句方框 + success**，工程上最合理的解释是：

**声明的逻辑字体 id / 字重，在执行端没有稳定映射到「含完整简中字形」的字体文件；发生静默回退或错配后，实际用于 `drawtext`（或等价路径）的 face 不含 CJK。**

常见断点包括（可多选叠加）：

1. **逻辑 id → 物理路径映射表**未覆盖 **`noto-serif-cjksc` + 300**，回退到默认英文字体。  
2. **`font_weight` 类型**：入参为字符串 **`"300"`**；若执行端只认整数或枚举名，**匹配失败**后走默认分支。  
3. **镜像内未安装**对应字重的 Noto 子集，与 **`list_fonts` 产品配置**不一致。  
4. **FFmpeg 仅用 `font=` 名字**、未强制 **`fontfile=` 绝对路径**，在精简环境里 fontconfig 解析到错误 face。  
5. **静默回退**：解析失败仍返回 `success: true`，观测上难以区分「真成功」与「用缺字字体画完」。

## 五、可落地的解决办法

### 1. 业务侧 / Agent 侧（快速验证）

- 同一套字幕，将 **`font` 改为 `noto-sans-cjksc`**，**`font_weight` 改为 `400`**（或目录默认），再导出；若方框消失，则强烈指向 **Serif/300 映射或安装** 问题，而非文案或「不能用宋体」。  
- **`font_weight` 与接口文档对齐**：优先传与 `list_fonts` 中 `weights` **同类型**的值（数字与字符串二选一并写死）。  

### 2. 执行端 / 工程侧（根治）

- **逻辑字体 + weight + style → 唯一 `fontfile` 路径**；解析失败应 **报错或重试**，禁止静默用拉丁字体画完。  
- 渲染时 **显式传入 `fontfile`**（若走 FFmpeg drawtext），避免仅靠名字解析。  
- **镜像与 `list_fonts` 对齐**：列出的每个 id/weight 必须在 worker 上存在对应文件。  
- **合成前自检**：用同一 `fontfile` 试排「测试」等汉字，失败则自动 fallback 到已验证的 **Sans 400** 并打日志。  
- **可观测性**：在 tool `output` 或日志中附带 **`resolved_font_path`**、**ffmpeg stderr 摘要**，便于下次从 Langfuse 直接定因。  

### 3. 规范侧（减少再犯）

- 在 **assembly-skill**（或字幕专用 reference）中约定：**简中烧录字幕默认 `noto-sans-cjksc` + 400**；若用户或风格明确要求宋体/细体，再选 Serif，并规定 **失败时降级策略**。
