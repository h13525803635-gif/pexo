# 项目 14848151814：画幅混用、字幕重叠与样式漂移问题说明

**Langfuse trace**：`analysis/langfuse-data/cases/14848151814/trace-1-07b98a04.json`（主执行 trace，约 1062s / 177 observations）  
**分析日期**：2026-05-07  

---

## 一、用户侧现象

- 同一条成片中，部分段落是标准横版观感，部分段落看起来接近方图（用户描述为「有的 16:9，有的 1:1」）。
- 画面上已经有大量文字说明，同时又出现字幕，信息明显重叠。
- 字幕（或字幕样式）在不同时间段视觉变化较大，观感不统一。

---

## 二、根因分析

### 核心问题：失败回退导致素材混搭 + 文案层与字幕层职责重叠 + 文本样式无统一模板

#### 2.1 画幅不一致：场景视频生成失败后，用静态图补位进入最终时间线

该 case 中 `video_generate` 多次请求 `aspect_ratio: "16:9"`，但部分场景被 credits 阻断（`billing_credit_blocked_info`），导致视频段未成功产出。  
进入 `video-editor__execute_edit_video` 时，时间线实际混用了两类素材：

| 素材类型 | 典型文件 | 进入时间线方式 |
|------|------|------|
| 成功视频段 | `vid3_1453.mp4`、`vid4_golden_age.mp4`、`vid5_decline.mp4` 等 | 直接作为 video clip |
| 静态场景图 | `scene1_founding.png`、`scene2_expansion.png`、`scene7_sick_man.png` | 以 `out_ms` 方式拉长为伪视频片段 |

静态图与视频段混剪后，即便容器输出是 1920x1080，用户仍会感知到构图比例不一致（“像 1:1”）。

#### 2.2 文字重叠：合成时写入大量 `text` 轨，语义与字幕层重复

两次主要合成调用（`ottoman_meme_animation`、`ottoman_meme_v2`）都在时间线上写入大量文本：

| 合成版本 | text clips 数量 | 特征 |
|------|------|------|
| 第一版 | 31 条 | 场景标题 + 叙事句 + 梗文案混合 |
| 第二版 | 20 条 | 仍包含大量叙事句，与字幕语义重叠 |

这些文本本质上承担了“字幕/旁白转写”职责。若播放器或平台再叠一层字幕，就会出现双层信息重叠。

#### 2.3 “字幕字体一直变”：并非频繁换字体家族，而是 clip 级样式频繁切换

从最终合成参数看，字体家族主要是 `montserrat`，但以下样式参数在 clip 之间频繁变化：

- `fontsize`
- `font_weight`
- `fontcolor`
- `bordercolor` / `borderw`
- `y`（文本垂直位置）

因此用户侧主观体验是“字幕一直在变”，实际是样式策略漂移。

## 三、问题本质

| 对象 | 真实对齐关系 |
|------|------|
| **画幅一致性** | 依赖场景视频是否成功；失败后静态图补位破坏一致性 |
| **文本信息层** | 画面 text 与字幕在同一语义层表达 |
| **字幕视觉规范** | 无全局样式基线，按 clip 临时调整 |

这不是单点参数错误，而是**生成失败回退策略 + 合成文本策略 + 样式策略**三个环节同时缺少硬性约束。

## 五、直接原因链

```
场景视频批量生成
  → 多个 video_generate 因 credits 阻断失败
    → 组装阶段仍继续执行
      → 失败场景改用 scene*.png 静态图补位
        → 与成功 vid*.mp4 混剪
          → 用户观感出现“16:9 + 1:1 混用”

同时：
  合成时写入大量 text clips（20~31条）
    → 语义与字幕层重复
      → 画面文字与字幕内容重叠

并且：
  text_opts 按条目频繁改 size/weight/color/stroke/y
    → 样式漂移
      → 用户感知“字幕字体一直在变”
```

## 六、修复建议

### 6.1 短期（工具链 / 执行层）

1. **场景完成率门禁**：必需场景 `video_generate` 未成功时，禁止进入最终合成。
2. **画幅一致性门禁**：合成前逐条 `ffprobe`，非 16:9 素材直接拦截。
3. **兜底图比例强制**：`image_generate` 兜底图必须显式传 16:9。

### 6.2 中期（Agent / skill 层）

4. **文本分层规则**：`title_card` 只保留章节标题；叙事内容交给 `subtitle`，禁止双层表达同一句。
5. **重叠去重规则**：字幕与画面文本语义高相似时自动去掉一层。
6. **安全区规则**：标题固定上方区域，字幕固定下方区域，禁止同区叠放。

### 6.3 长期（产品规范）

7. **统一字幕样式模板**：固定字体、字号区间、描边和位置，仅允许受控缩放。
8. **样式漂移检测**：对样式 tuple 数量设阈值，超限直接阻断出片。
9. **失败回退策略升级**：由“静态补位继续出片”改为“显式失败并提示重试/补资源”。
