# 项目 14848151814：画幅混用与字幕重叠及样式漂移问题说明

**Langfuse trace**：`analysis/langfuse-data/cases/14848151814/trace-1-07b98a04.json`（主执行 trace，约 1062s / 177 observations）  
**分析日期**：2026-05-07  

---

## 一、用户侧现象

- 成片里部分段落看起来是横版 `16:9`，另一些段落观感接近方图 `1:1`（或明显非统一画幅）。
- 画面上已经有大量文案时，字幕（或播放器侧字幕）再叠加，出现明显信息重叠。
- 用户主观感受是“字幕字体一直在变”，整体阅读风格不稳定。

---

## 二、根因分析

### 核心问题：场景素材成功率不一致 + 文字层职责混乱 + 样式缺少统一基线

#### 2.1 画幅混用：`video_generate` 失败后回退到静态图，导致素材来源混搭

同一轮里，`video_generate` 虽然请求了 `aspect_ratio: "16:9"`，但多段关键视频调用被 credit block 中断，输出中出现 `billing_credit_blocked_info`。  
随后在 `video-editor__execute_edit_video` 组装时，时间线混用了两类素材：

| 素材类型 | 示例 | 特征 |
|------|------|------|
| 视频段（成功生成） | `vid3_1453.mp4`、`vid4_golden_age.mp4`、`vid5_decline.mp4` 等 | 明确是视频片段，按 16:9 输出 |
| 静态场景图（失败回退） | `scene1_founding.png`、`scene2_expansion.png`、`scene7_sick_man.png` | 原始图像来源与构图不一致，放入 1920x1080 时间线后出现“像 1:1”的观感 |

也就是说，这不是单一参数填错，而是**失败后降级路径把静态图和动态视频混剪**导致的画幅一致性破坏。

#### 2.2 文案重叠：时间线上堆叠了大量 text clip，和字幕语义高度重复

在两次主要合成调用（`ottoman_meme_animation`、`ottoman_meme_v2`）中，`video-editor__execute_edit_video` 都写入了大规模 `kind: "text"` 轨道：

- 第一版约 31 条文本 clip
- 第二版约 20 条文本 clip

这些文本本身就是叙事句（例如场景说明、吐槽文案、口语化台词），与“字幕本应承担的旁白转写信息”是同一层语义。  
因此只要播放器/平台再叠一层字幕，就会出现“画面文字和字幕打架”。

#### 2.3 “字幕字体一直变”：字体家族基本一致，但样式参数频繁漂移

从最终合成参数看，主字体家族大多是 `montserrat`，但每条文本 clip 的样式参数变化很大：

- `fontsize` 多档变化
- `font_weight` 多档变化
- `fontcolor` / `bordercolor` / `borderw` 频繁切换
- `y` 位置上下跳动（顶部/底部不同锚点）

最终并非“换了很多字体家族”，而是**同一字体在不同 clip 上被大量改样式**，用户侧感知为“字体一直在变”。

## 三、问题本质

| 对象 | 实际状态 |
|------|---------|
| **画幅一致性** | 依赖场景视频是否成功；失败后用静态图兜底，导致观感不一致 |
| **文字信息层** | 画面 text 与字幕承担了同一叙事职责，造成重复 |
| **字幕视觉统一性** | 无全局样式约束，按 clip 自由改参数 |

本质上是同一条产线的三个“缺少 gate”问题：  
**素材一致性 gate 缺失、信息分层 gate 缺失、样式规范 gate 缺失**。

---

## 四、直接原因链

```
场景视频批量生成
  → 部分 video_generate 因 credits 阻断失败（credit blocked）
    → 组装阶段仍继续执行
      → 失败场景用 scene*.png 静态图补位
        → 与成功的 vid*.mp4 混剪
          → 画幅/构图观感不一致（用户感知“有的像 1:1”）

同时：
  大量 text clip 写入时间线（20~31条）
    + 平台/播放器字幕层并存
      → 信息语义重复、视觉区域重叠

并且：
  text clip 按条目单独改 size/weight/color/stroke/y
    → 视觉风格漂移
      → 用户感知“字幕字体一直在变”
```

## 五、修复建议

### 5.1 短期（工具链与执行门禁）

1. **加“场景完成率门禁”**：任何必需场景 `video_generate` 失败时，禁止直接进入最终合成。
2. **加“画幅校验门禁”**：合成前对每个入轨素材做 `ffprobe`，非 `16:9` 则拒绝合成（或走明确裁切策略）。
3. **统一 image_generate 比例**：静态兜底图也必须显式传 `16:9`，避免默认方图进入时间线。

### 5.2 中期（Agent / skill 规则）

4. **文字层职责拆分**：  
   - `title_card`（少量标题卡）  
   - `subtitle`（旁白转写）  
   两者不能在同时间窗表达同一句语义。
5. **重叠去重规则**：若字幕与画面文案语义相似度高于阈值，自动抑制其中一层。
6. **安全区约束**：标题固定上方区，字幕固定下方区，禁止同区叠放。

### 5.3 长期（产品规范化）

7. **字幕样式单一源（Style Preset）**：固定 `font / weight / color / stroke / y`，只允许“长句小幅缩放”一种受控变化。
8. **样式 lint**：统计全片字幕样式 tuple 数量，超阈值直接拦截产出。
9. **失败回退策略重构**：从“静态图硬补位”改为“失败即中止并提示重试/补款”，避免产出低一致性成片。

## 六、建议落地优先级

| 优先级 | 事项 |
|------|------|
| **P1** | 场景完成率门禁 + 画幅校验门禁（不满足即不出片） |
| **P1** | 文案层与字幕层去重 + 区域避让 |
| **P2** | 全局字幕样式模板 + 样式 lint |
| **P2** | 失败回退策略由“降级出片”改为“显式失败” |
