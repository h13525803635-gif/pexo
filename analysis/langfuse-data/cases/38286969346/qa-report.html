Case 38286969346 — 首次成片字幕超出安全区

Langfuse trace: trace-1-15c5c7f0.json · 成片 corporate_tax_ad_final · 720×1280 · 2026-05-04

现象：首版成片底部英文字幕在观感上超出平台/画幅常用安全区（横向出画或贴底）。

根因（trace 可核对）

  • 横向：execute_edit_video 使用 kind: "text"，首句整段英文单行 + fontsize: 30，在 720 宽下估算 text_w > w，x: (w-text_w)/2 无法把整行收进安全宽度。
  • 纵向：统一 y: h-220 为与内容行高脱钩的固定像素，多行或描边时易显贴底；竖版 9:16 常需更大底留白。
  • 数据源：TTS 已返回 subtitle_file，成片用手写多段 text 时间窗，未用 cue 级真源。

解决方法（执行清单）

项做法
横向超长句拆多条 clip 或降字号；保证单行宽度 < 约 0.84w～0.88w。
纵向禁止全局 h-220 / h-130；改用 y: h-text_h-h*0.10（或等价底留白 12%～18% 画高）。
时间轴优先用 subtitle_file 的 cue 起止 + 各段 VO 的 start_ms 换算绝对时间。
轨类型产品支持时优先 kind: "subtitle" + 规范 margin_v；须满足渲染器对 out_ms 等字段的要求。

证据摘录（edit_spec）

"kind": "text",
"text_opts": {
  "fontsize": "30",
  "x": "(w-text_w)/2",
  "y": "h-220",
  ...
}
