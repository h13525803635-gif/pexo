# 项目 04453569710：人物出镜口播音画不同步问题说明

**Langfuse trace**：`analysis/langfuse-data/cases/04453569710/trace-4-c70234bc.json`（主执行 trace，636s / 120 observations）  
**分析日期**：2026-05-07

## 一、用户侧现象

- 视频中女性数字人正对镜头「说话」（嘴部有张合动作），但**口型与旁白音频节奏完全不同步**。
- 听感上旁白说到某句时，画面中人物嘴型对不上该句文案。

---

## 二、根因分析

### 核心问题：视频与音频是独立生成再后贴的，口型无法对齐

#### 2.1 视频生成：`sound: "off"` + 无 audio_list — 纯视觉口播动画

Agent 最终成功生成的两段视频（`avatar_seg1`、`avatar_seg2`）参数如下：

| 参数 | seg1 | seg2 |
|------|------|------|
| provider | Seedance | Seedance |
| model | doubao-seedance-2-0-260128 | doubao-seedance-2-0-260128 |
| mode | reference2video | reference2video |
| sound | **off** | **off** |
| audio_list | **无** | **无** |
| duration | 15s | 15s |

prompt 中写了 `"mouth moving as if actively explaining something"`，模型生成了**视觉上嘴部张合的动画**，但这只是模型自由发挥的「假说话」动作，**没有任何真实音频信号驱动口型**。

ffprobe 确认：两段视频均 **`nb_streams=1`**（仅视频流，无音频流）。

#### 2.2 Agent 曾尝试 audio_list 驱动口型但全部失败

Agent 实际尝试了将 TTS 音频（`audio_list`）喂入 Seedance 做口型锁定，但连续失败了 **3 次**：

| 尝试 | 失败原因 |
|------|---------|
| 第 1 次 | AAC 格式 → `"audio format specified in the request is not valid"` |
| 第 2 次 | AAC 格式 → 同上（因 video-editor 输出始终为 AAC） |
| 第 3 次 | 原始 MP3 30s → `"audio duration must be ≤ 15.2s"` |

**Agent 无法将 MP3 切为 ≤15s 的 MP3 片段**（video-editor 只输出 AAC），最终放弃 audio_list，改为 `sound: "off"` 无音频输入生成。

#### 2.3 合成阶段：视频轨 volume=0，独立叠旁白

最终合成 `onboarding_final`：

```json
{
  "tracks": [
    {
      "kind": "video",
      "clips": [
        {"source": {"file": "avatar_seg1", "out_ms": 15042}, "volume": 0},
        {"source": {"file": "avatar_seg2", "out_ms": 15042}, "volume": 0}
      ],
      "transitions": [{"type": "cut"}]
    },
    {
      "kind": "audio",
      "clips": [
        {"source": {"file": "原始VO_30s", "out_ms": 30197}, "volume": 1.0}
      ]
    }
  ]
}
```

- 视频轨 `volume: 0` → 正确关掉了视频自带音频（本 case 中视频本无音频，但设置正确）。
- 音频轨单独放原始 VO → **旁白的文案、节奏、停顿均与视频画面中数字人的嘴型无关**。

## 三、问题本质

| 对象 | 对齐的是 |
|------|---------|
| **旁白音频** | 用户上传的 30s MP3（真实音频） |
| **数字人口型** | Seedance 模型自由生成的动画（prompt 仅写"嘴在动"） |
| **两者关系** | **完全独立，无因果耦合** |

这不是双音轨叠加问题（96933223677 case 的根因），而是**口型驱动缺失**问题：音频从未被用于驱动视频生成。

## 五、直接原因链

```
用户上传 30s MP3
  → Agent 切分音频为 2×15s 片段
    → video-editor 输出为 AAC 格式
      → Seedance r2v 不接受 AAC → ❌ 失败
    → 改用原始 MP3 整段传入
      → 30s > 15.2s 上限 → ❌ 失败
    → 放弃 audio_list
      → sound: "off" + prompt 描述嘴动 → 纯视觉动画
        → 合成时贴上原始 VO
          → 音画不同步 ✓ 复现
```

## 六、修复建议

### 6.1 短期（video-editor / 工具链层面）

1. **支持 MP3 输出格式**：video-editor 的音频切分应能输出 MP3（或 WAV），而非仅 AAC。Seedance r2v 的 audio_list 仅接受 MP3/WAV。
2. **或在 video_generate 侧做音频格式转换**：在调用 Seedance API 前自动将 AAC → MP3。

### 6.2 中期（Agent / skill 层面）

3. **generation-skill 应强制执行 audio_list 驱动**：当场景为「口播 / talking head」且已有 TTS/VO 时，**不得 fallback 到无 audio_list 的 sound-off 方案**。若 audio_list 失败，应报错并请求用户介入，而非静默降级。
4. **音频切分策略**：30s VO → 2×15s 时，确保切分后的每段 ≤15.2s 且格式为 MP3。可在 Agent 侧先 ffprobe 获取时长，精确计算切分点。

### 6.3 长期（产品层面）

5. **口播场景标准化产线**：
   - `VO 音频` → 按 15s 切分为 MP3 片段
   - 每段 `video_generate` 必须带 `audio_list` + `sound: "on"`
   - 合成时视频轨 `volume: 0`，音频仅来自 VO 轨
6. **若 Seedance r2v 的口型驱动质量不够**：考虑专用 lip-sync 后处理方案。
