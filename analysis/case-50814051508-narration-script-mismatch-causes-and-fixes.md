# 案例 50814051508：口播与用户提供文稿不符 — 原因与解决办法

**项目 / 会话 ID**：50814051508  
**分析依据**：Langfuse 生产 trace（如 `pexo:50814051508`，含 `video_generate` 入参与策略说明）  
**文档日期**：2026-04-20  

---

## 一、现象（用户侧描述）

- 成片中的口播与用户提供的文稿**整体对不上**，听感上像「完全不是同一份稿」。  
- 除大段内容不一致外，还存在**个别单词念错 / 念成别的词**的情况。  

---

## 二、根本原因（技术链路）

### 1. 口播来源不是「按稿逐字配音」

成片使用的是 **视频模型声画共生成**（例如 Kling `reference2video` + `sound: on`），对白写在 `video_generate` 的 prompt 里，由模型在生成画面时**顺带生成语音**。  

该路径的**设计目标**是「自然口语 + 口型大致同步」，**不是**「与用户上传文档逐字一致」。因此不能与「把全文交给 TTS 严格朗读」的预期混为一谈。

### 2. Agent 在调用模型前已改写、压缩文稿

在进入模型之前，系统已将长稿拆成多段（约每段 9–10 秒），并做了：

- **删除或弱化**稿中的**主持人问句**等结构，改为更短的「卖点式」口语；  
- **缩写、同义改写**大段回答，只保留部分信息；  
- 对稿中**重复段落**做合并，顺序与字面稿不完全一致。  

因此：**成片念的文本在输入侧就已经不是用户原文**，用户听到的差异不仅是「模型念歪」，更是「压根没把全文送进该段生成」。

### 3. 共生成语音对 prompt 内台词仍可能漂移

即便 prompt 里用引号写了目标句子，多模态模型生成的语音仍可能出现：

- 同义替换、省词、语序微调；  
- 专名、品牌、数字表述不稳定（例如价格在 prompt 中写作 *twelve ninety-nine* 与稿面 *$12.99* 在字面上即不一致）；  
- 偶发**音近错词**或吞字，听感为「念错单词」。  

上述与第 2 点叠加，会表现为**大段不符 + 个别词错误**。

### 4. 与「用户是否写固定口播」的关系

用户未明确写「固定口播 / 必须逐字 / 不得改稿」时，更容易被路由为**出镜对白共生成**。  
但即便用户写了，若 **skill / 路由未强制「verbatim 模式」**，仍可能出现第 2、3 点的问题；**明确指令有助于触发正确流程，但不是唯一条件**，还需要产品规则与实现兜底。

---

## 三、解决办法

### A. 用户侧（立即可做，降低偏差）

1. **在需求里写死口径**（建议原文粘贴到消息中）：  
   - 例如：「口播必须与下列英文**逐字一致**，不得增删改；不得省略问句；数字与品牌按原文朗读。」  
2. **把可念文本单独成块**：问句一行、答句一行，或按镜头分段编号，避免一大段散文被模型自动摘要。  
3. **敏感词与专名**：对易错品牌名、数字，单独一行说明「必须读成：…」或提供音标/拆分读音（若后续走 TTS 更易控制）。  

### B. 产品 / Agent 策略（从根上对齐「文稿 = 声轨」）

1. **检测「用户已提供完整口播稿」**时，默认或显式选项：  
   - **verbatim 模式**：对白不走视频共生成随意念，改为 **分段 TTS（或已支持的严格朗读管线）** + 与画面合成；或对「必须口型同步」的镜头单独约定技术方案。  
2. **路由规则**（对应 creative-skill / generation-skill）：  
   - 当用户稿为「主持问 + 答」结构时：**问句与答句要么都进锁定台本，要么都删**，禁止只保留答句却让用户以为会问句也会念出。  
3. **生成前校验**：将即将送入 `video_generate` 的引号内台词与用户上传段落做 **diff 或长度/句数检查**；若偏离超过阈值，**中断并让用户确认**「摘要版」或「逐字版」。  
4. **数字与货币**：统一策略（始终 `$12.99` 或始终口语 `twelve ninety-nine`），并在 UI 或 skill 中写清，避免用户认为「念错」实为「系统改写表述」。  

### C. 质量与运营

1. 对「教育 / 答辩 / 合规」类视频，在模板中默认勾选 **「口播与上传文件一致」**。  
2. 发布后若仍有个别错词，走 **modification-skill** 仅重生成有问题片段的音频或整段，并强制使用锁定文案。

## 四、小结表

| 层级 | 原因 | 对策方向 |
|------|------|----------|
| 模型能力 | 声画共生成不保证逐字 | 逐字场景改 TTS / 锁文案管线 |
| Agent 行为 | 摘要、删问句、按时长砍内容 | verbatim 检测 + 生成前确认 + skill 约束 |
| 用户输入 | 意图未写清 | 明确「不得改稿」+ 结构化分段 |
| 听感 | 数字/专名表述不一致 | 统一书写与朗读规范 |
