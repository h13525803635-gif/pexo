## 现象

用户上传 7 张配图 + 7 分钟 YouTube 脚本，期望成片含动态画面；实际经历两条路径：

1. **路径 A**：静图 + VO + BGM → `execute_edit_video` → 成片 `5_signs_new_website_full.mp4`（约 5 分钟）✅  
2. **路径 B**：用户要求 montage / stock footage 后，8 段 `video_generate` 仅 2 段成功，其余积分不足，**未组装** ❌

## 解决方案

### 用户侧

- 明确写：「用上传图做 image-to-video，不要纯静图铺轨。」  
- 积分有限时：「关键段 video_generate + 其余 uploaded 静轨 + 必须 execute_edit 交片。」

### Skill / Agent

1. 有 `<original-image>` 且要动效 → 禁止仅以 PNG 铺轨交片；`image_list` 必须含对应 uploaded 路径；产品段用 `image2video`。  
2. 强制 subject-asset → generation → assembly 顺序。  
3. `insufficient_credits` 后：降级混合成片（已有 vid + uploaded + VO），禁止无成片结束。  
4. 并行 batch 前做积分预算，避免一次 8 段 video_generate。

### 产品

- 上传时选项：静图幻灯片 / 图生动态 / 全 AI 重生成。  
- `reference2video` 且 `image_list` 为空、项目有 uploaded → 拦截或警告。

### 本案重做顺序

```
uploaded 映射 → video_generate(image2video + image_list)
  → 复用 vo → bgm → execute_edit_video → show_final_video
```
