## 二、解决方法（按优先级）

### A. 声音与字幕铺满整片（推荐）

1. **按段或按句拆 TTS**（或与镜头对齐的 VO 分轨），使 **音频 + 烧录字幕** 覆盖 **0～成片总时长**（本 case 约 20s），避免「后半段真空」。  
2. 若坚持 **仅后期一条人声**：要么 **缩短总片长 ≤ VO 时长**，要么 **放慢/重复/补环境**（需符合产品规范），不能 20s 画 + 10.5s 声却无设计。  
3. **可见主持 + 固定文案**：考虑 **VO 分片 + `audio_list` 驱动 Seedance`**（每段 ≤15s），口型与声同源，再统一混音策略。

### B. 减少跳切与段间不一致

1. **能单段则单段**：目标 ≤15s 时优先 **单次 Seedance**，避免无必要的双段 + `cut`。  
2. **必须双段时**：评估 **serial + `video_list` 延续**（若产品与模型支持），或 **同一批次生成两段**、避免只重跑其中一段。  
3. **Prompt**：减少强分段 `[x–ys]` 堆砌，或改为 **更连贯的单段动作描述**，降低段内「假跳切」。  
4. **参考图**：主持人与 UI **统一目标纵横比**（例如都先收到 **4:3 或都 16:9 安全框**），再进 ref2video，减轻「拼贴感」。

### C. 画幅策略（避免 4:3 需求与 16:9 出片反复横跳）

1. 需求为 **4:3** 时，在规划阶段就假定 **中间产物 16:9**，构图按 **中心 4:3 安全区** 设计，减少后期裁切带来的再构图。  
2. 或 **交付 16:9** + 黑边/裁切二选一，由用户确认，避免未告知的裁切。

### D. 技能 / Agent 规则侧（防回归）

1. **`sound: off` + 全片 VO**：装配表中必须校验 **`sum(视频时长)` 与 `VO 时长` 与字幕轨** 一致，否则 **拦截交付** 或自动补静音/补轨策略。  
2. **多段成片**：若一段重跑，**强制评估是否需重跑相邻段** 以保持连贯。  
3. **Seedance `aspect_ratio`**：调用前校验枚举，**禁止直接传 4:3** 到不支持 API。
