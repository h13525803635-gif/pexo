# Case 75600282409（Nama HR Policy）混音问题说明与方案

- **项目 / 会话 ID**：75600282409  
- **Langfuse**：`pexo:75600282409`（如 `trace-2-46363568.json` 含成片 `execute_edit_video` 参数）  
- **摘要**：全片有独立 TTS 旁白 + 企业 BGM，同时 6 段 `video_generate` 均为 **`sound: "on"`**，画面内环境声与 VO、BGM 三路叠加；BGM 为固定电平、无旁白闪避，易出现人声被糊住、频段打架、听感脏。

---

## 一、问题（现象 → 根因）

### P1：画面原声 + 旁白 + BGM 同时进混音（最严重）

| 根因 | 说明 |
|------|------|
| **生成阶段** | 6 段 Seedance `video_generate` 全部为 **`sound: "on"`**；提示虽写「无配乐」，模型仍会输出环境声 / 房间底噪等。 |
| **组装阶段** | `execute_edit_video` 中 **video 轨 clip 未压低或关闭画面内音频**（仅 `speed` 等），再叠 **VO `volume: 1.0`** 与 **BGM `volume: 0.18`**。 |
| **听感** | 典型 **三路并行**：环境底噪 + 全片旁白 + 企业 BGM → 掩蔽加重、清晰度下降。 |

### P2：BGM 全程固定电平，无旁白闪避（ducking）

| 根因 | 说明 |
|------|------|
| **参数** | 三段成片里 BGM 均为 **固定 `0.18`**，仅 `fade_in` / `fade_out`，**无 VO 时段再压低**。 |
| **听感** | HR 类旁白密集时，BGM 再干净也容易与 VO **抢存在感**；在 P1 已叠画面声时更明显。 |

### P3：整体响度目标无法单独解决掩蔽

| 根因 | 说明 |
|------|------|
| **`target_lufs: -14`** | 有利于成片整体响度一致，**不减少层数、也不自动给 VO 让频**，不能替代层级与避让策略。 |

### 参考：本 case 成片轨道结构（与三段一致）

- **Video**：2 个 clip + `crossfade` 500ms，无画面轨音量衰减。  
- **Audio 1**：两条 VO，`start_ms` 首尾相接，`volume: 1.0`。  
- **Audio 2**：单条 BGM，`volume: 0.18`，带淡入淡出。

---

## 二、解决方案

### A. 允许关画面声时（对照，非本问首选）

- 有独立 VO 的片子：`video_generate` 使用 **`sound: "off"`**，或组装时对画面轨 **mute**，只保留 VO + BGM（必要时单独铺极轻 room tone）。  
- 与「不能关原音」场景互斥，仅作策略对照。

### B. 不能关原音时的可行方案（按落地难度）

#### B1：旁白时段闪避 BGM（ducking）— 优先推荐

- **做法**：在 **VO 活跃时段** 将 BGM 再压低（自动化包络 / 侧链思路，视 `execute_edit_video` 能力而定），VO 间隙再缓抬。  
- **效果**：不动画面原声，显著减轻「BGM + VO」掩蔽；与 assembly 范例中「VO + BGM + ducking」一致。

#### B2：压低画面轨音量（不是关掉）

- **做法**：若 API 支持 **video clip 的 `volume` / `gain`**，将画面环境声置于 **明显低于 VO** 的一层（例如 VO 1.0、画面声 0.2～0.35 量级再微调）。  
- **效果**：保留「有空间感」的原声，但 **以可懂度为优先的层级**，减少与旁白抢能量。

#### B3：BGM 选曲与铺陈 — 少占中频、少铺满

- **做法**：选 **更疏、中频更干净** 的 corporate bed；或在旁白密集段 **拉长淡出 / 暂mute BGM**，仅在段间铺一点。  
- **效果**：不改原声，靠 **减少 BGM 频谱占用** 换 VO 清晰度。

#### B4：EQ / 分轨处理（依赖引擎能力）

- **做法**：对 BGM 或画面铺底做 **高通 / 略压中频**，给旁白留出 **2–4 kHz 左右存在感频段**（需 per-track EQ 支持）。  
- **效果**：减轻「糊」，实现成本取决于编辑器能力。

#### B5：处理旁白链

- **做法**：VO **轻度压缩** + 适度 **临场感 EQ**（注意齿音），使轻声仍清晰、整体更易「压过」环境噪声。  
- **效果**：在总线电平不变前提下提升可懂度；避免过度导致刺耳。

#### B6：时间维度避让

- **做法**：**有 VO 的句子时段** BGM 再低一档；**无 VO 的纯画面间隙** 再抬 BGM。  
- **效果**：原声与 BGM 不必全程「同时满」，听感更干净。

---

## 三、原则小结

不能关原声时，核心是 **层级**：**VO 最前 → 画面环境声中间 → BGM 最后**；手段上优先 **BGM ducking +（若支持）画面轨整体衰减**，再辅以 **选曲/留白** 与 **EQ / VO 处理**。

---

## 四、技能 / 规范侧（防回归，可选）

1. **有独立 VO 且保留画面声时**：assembly 默认建议 **BGM ducking** 或 **VO 段 BGM 包络**，避免固定系数一条轨铺到底。  
2. **generation**：若产品强制 `sound: on`，在 assembly 侧明确 **画面轨建议 gain 范围**，与 VO 做 **分级混音表**。  
3. **交付前检查**：关键句试听 **VO 可懂度**；若仍糊，按 B1→B2→B3 顺序加严。
