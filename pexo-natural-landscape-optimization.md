# Pexo 自然风景类视频 — 优化与提升汇总

> 本文汇总对 Pexo 制作**自然风光 / 风光大片**类视频的能力评析、主要弱点与可落地的提升路径（含使用侧与产品·Skill·工程侧）。  
> 保存位置：`~/Desktop/skill/pexo-natural-landscape-optimization.md`

---

## 一、能力总评（脱离具体项目）

- **架构匹配度**：多段生成 + 风格参考 + 环境声 co-gen + 后期 BGM/组装的路线，与自然类「无对白、靠空间与情绪」的片型**一致**。
- **强项**：预告片式分段节奏、SFX 与画面对位、Seedance 系在少人脸场景下的**质感与运动**、技能链（creative → generation）把意图落成可执行 prompt。
- **边界**：更接近**高完成度商用自然短片 / 频道包装**，而非默认达到「强地理识别度 + 影院级母带听感」；差距在**母题同质化、跨段听感缝合、工程链路稳定性**，不单是「画面不够好看」。

---

## 二、主要弱点（按类型）

| 弱点 | 表现 |
|------|------|
| 地理与作者记忆点 | 易落入泛化「史诗自然」，缺少可识别的具体地貌与行为 |
| 听感连续性 | 分段各自好听，整条片环境底噪/混响/电平易「换房间」 |
| 工程与组装 | `edit_spec` 混入非 HTTPS、`/workspace/` 路径或临时域名导致剪辑失败 |
| 音乐长弧线 | 长时长、复杂 composition 时重试多，影响情绪线稳定 |
| 节奏与片型 | 产品默认偏预告/广告分段；单镜长时间「酝酿」需 brief 与 prompt 主动约束 |

---

## 三、提升路径汇总

### 1. 地理真实感与差异化（Brief + 创意）

**使用侧**

- Brief 写**可验证具体性**：地貌、季节、晨昏、局地天气行为；少堆「国家地理、IMAX、epic」等泛词。
- 提供**锚定参考图**（光线、色彩、地貌贴近目标成片），让 reference2video 有明确锁，而非纯文本想象。
- 系列片：使用技能中的 **Visual Style Convention / Brand Visual Language Pack**，跨片复用四常量（渲染风格、色板、光线气质、颗粒质感）。

**Pexo / Skill 侧**

- **creative-skill**：对「纯风光、无用户图」增加**地理与气候锚**强制输出；禁止仅靠泛词进入高成本生成。
- 可选：挂载**地貌与季节光线**类知识片段，减少「全球都像同一套冰岛」的随机性。

---

### 2. 听感：从「段好听」到「一条母带」

**使用侧**

- 明确 **shared ambient foundation**（全片同一海风频段、同一空间混响性格等），各段 SFX 在其上叠加。
- 验收时重点听**段与段拼接处**（电平、底噪、混响尾）。

**Pexo / Skill 侧**

- **generation-skill**：落实 **SFX coherence plan**，handoff 中写死的共享底床需**逐段可复制进 prompt 首部**。
- **组装后**：产品层增加**轻量母带**（统一响度、必要时简单 EQ）以减少硬切听感。
- 若架构允许：导出或保留 **environment bed 单独轨**，便于后期精修。

---

### 3. 工程链与剪辑稳定性

**Pexo / 工程侧（高优先级）**

- **execute_edit_video** 前：**URL 规范化**（仅 HTTPS；workspace 路径须先解析/签发到项目 OSS；拒绝不可依赖的中转域名）。
- **单一真相源**：以生成完成后的 **canonical 资产 URL**（如项目 OSS 签名链）写入 `edit_spec`，避免插件临时 URL。
- **失败容错**：同一资源存在**多条 signed URL** 时，自动 failover 重试。

**使用侧**

- 大组装前对关键 clip 做**可访问性检查**（避免整轨失败后大面积返工）。

---

### 4. 音乐与情绪弧线

**Pexo / Skill 侧**

- **music_generate**：控制首轮复杂度；schema 失败时**降级**（更短 duration / 更简单 style 描述）再迭代。
- 坚持 **video-first** 时：先定剪接点与起伏，再让 BGM 跟片，减少音画节拍错位返工。

**使用侧**

- 在稳定性优先时，**略短、略简单的 BGM 规格**往往优于一次铺滿超长史诗参数。

---

### 5. 节奏与「慢大片」诉求

**使用侧**

- 需要**少切、长酝酿**时：在 brief 中写明；单段内用**时间轴子句**拉长同空间内变化，而非盲目增加段数。
- 接受总时长由多段构成时，在组装阶段用**动作/光线/运动模糊**藏剪接。

**Pexo / Skill 侧**

- **creative-skill**：对「慢节奏风光」给出**少分镜、单段内时间轴细分**的默认策略。
- 可选模板：**单段上限内的伪一镜**（prompt 内限制硬切、以缓推缓摇为主）。

---

## 四、投入产出优先级（建议执行顺序）

1. **Brief + 锚图**：具体地理气候锚 + 锁光锁色参考图 —— 对「像不像一条好风光片」性价比最高。  
2. **共享环境声床 + 组装后轻母带** —— 对「像不像成片」性价比最高。  
3. **资产 URL 规范化与失败切换** —— 对「能否稳定交付」性价比最高。
