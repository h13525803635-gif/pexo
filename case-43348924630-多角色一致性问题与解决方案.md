# Case 43348924630 — 多角色一致性问题与解决方案

**需求**：在冰雪奇缘主题生日视频基础上，将「笛笛」替换为真实女孩「陈琳夕」（5岁，爱莎公主造型），保留安娜、艾莎、雪宝等原版角色  
**时间**：2026-04-20 11:46–16:26（约 4.5 小时，9 个 trace）  
**生成量**：20+ 版视频（linxi_sing_v1~v4、linxi_candles_v1~v6、早期 didi_birthday v1~v8）

---

## 问题描述

每次为修改陈琳夕形象重新生成视频时，安娜、艾莎、雪宝的外貌以及蛋糕、桌子的样式都随之漂移，无法与前一版保持一致。用户多次反馈：

- 「凭空多出来别的人物主体」（Trace-8）
- 安娜辫子粗细/脸型/服装细节版版不同
- 艾莎发色深浅、面部轮廓随机变化
- 蛋糕形状、花朵位置、蜡烛数量每次不同

---

## 根因分析

### 1. 配角和道具没有视觉锚定，全靠文字描述

所有 20+ 次 `video_generate` 对配角的控制方式始终如下：

```
reference_images = [chenlinxi_ref.png]   # 只传了陈琳夕参考图
prompt = "Anna (red-brown braids, magenta dress), Elsa (platinum side braid)..."
```

原始视频（`a_nTzC4t_...mp4`）里已经有安娜/艾莎/雪宝/蛋糕/桌子的具体视觉形象，但**从未被截帧作为 reference image 传入任何一次 video_generate**。Kling 每次从文字描述重建整个场景，一致性无从保证。

Kling `reference2video` 支持最多 7 张 reference images，整个项目始终只用了 1 张（陈琳夕参考图），剩余槽位全部浪费。

### 2. 陈琳夕参考图本身不稳定，带动场景风格漂移

| Trace | image_generate 次数 | 修改内容 |
|-------|-------------------|---------|
| Trace-3 | 18 次 | 从刘海+麻花辫改成丸子头 |
| Trace-4 | 10 次 | 调整眼睛大小 |
| Trace-5 | 6 次 | 调整表情严肃度 |
| Trace-8 | 6 次 | 调整肤色、眉毛、发色 |

参考图每次都在变化，其风格、色调、光照的微差会影响 Kling 对整个场景的风格解读，配角也跟着漂。

### 3. 多角色场景稀释主角控制力

4 个角色同框时，Kling 在角色间分配 attention，主角陈琳夕的外貌遵循度进一步下降，同时模型无法稳定控制角色总数。

### 4. 真实照片作为 reference 遇到错误，未走通

Trace-7 尝试「替换成真实人脸」时出现 `Image format is invalid` 报错和 15min poll timeout，未能成功。

### 5. 项目初期未建立「保留元素」视觉档案

Agent 第一次响应只确认了「笛笛的照片」和「格式」，未从原视频提取配角帧作为视觉基准。modification-skill 的确认门只覆盖了「要改的元素」，缺少对「要保留的元素」的显式锁定。

---

## 解决方案

### 层次一：生成策略（最直接，当前工具已支持）

从原视频截取配角和道具的关键帧，作为 reference images 一并传入：

```python
reference_images = [
    chenlinxi_ref.png,      # 主角（要改的）
    frame_anna.png,         # 安娜截帧（要保留的）
    frame_elsa.png,         # 艾莎截帧（要保留的）
    frame_cake_table.png,   # 蛋糕+桌子截帧（要保留的）
]
reference_video = original_video  # 整体风格/运镜锚定
```

### 层次二：modification-skill 加入 Phase 0「保留元素建档」

在进入生成前增加一步：

1. 用 ffmpeg 从原视频抽取关键帧
2. 识别画面中需要「保留」的元素（配角、道具、场景）
3. 将这些帧保存为 reference images，建立视觉档案
4. 向用户展示确认：「以下元素我将保持原样 [附图]，请确认」

### 层次三：用户确认门（UX 层面）

对「局部替换」类请求，开口生成前明确告知：

> 「我理解你想把笛笛换成陈琳夕，其余角色（安娜、艾莎、雪宝）和场景（蛋糕、桌子）将保持原视频风格。我会从原视频截取参考帧来锁定外貌，确认吗？」

当前 modification-skill 的 Format Gate 只确认了「改什么」，缺少对「保留什么」的显式确认。

---

## 优先级建议

| 方案 | 改动成本 | 效果 |
|------|---------|------|
| 自动截帧 + 塞满 reference_images | 低（工具已有） | 最直接，立竿见影 |
| Phase 0 保留元素建档 | 中（skill 逻辑改动） | 根本性修复 |
| 用户确认门 | 低（prompt 改动） | 防止大量返工 |
